What you get on day one

No SDK to integrate, no agent code to change. Trabecc speaks plain MCP on both sides — your existing client points at it, your existing servers run behind it.

Default-deny policy

Glob-matched allow/deny rules with optional argument-level predicates. Hide tools from the catalog or block specific args (no fs__write_* under /etc).

Audit log on every call

SQLite-backed, ships with the binary. Records the calling client, redacts credential-shaped args, prunes itself. Cloud retention if you want it.

Per-tool rate limits

Token bucket per qualified tool name. Catch the runaway agent before it ships your API budget to a stranger.

Fan-out across servers

Connect filesystem, GitHub, Slack, Postgres — present them to your agent as one MCP endpoint with namespaced tools (github__search_issues).

Live dashboard

Server-rendered HTML at localhost:4577. Recent calls, outcome breakdown, top tools. Auto-refresh, no JS required.

One config file, zero build

YAML configures servers, rules, and rate limits. One Node binary. Self-host on a laptop or run in your fleet — same code.

30 seconds to a governed agent

Drop Trabecc between Claude Desktop / Cursor / Claude Code / your custom agent and the MCP servers it already uses. Policy lives in a YAML file you can grep.

# trabecc.yaml
defaultPolicy: deny

servers:
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/code"]

rules:
  - match: "filesystem__read_*"
    effect: allow
  - match: "filesystem__write_*"
    effect: deny
    when: { path: "/etc/*" }
    reason: "no writes under /etc"

rateLimits:
  - match: "filesystem__*"
    perMinute: 120
